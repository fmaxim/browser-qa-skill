Bug 1: Hamburger button too small (20x20px, should be 44x44)

Home | About | Contact

The hamburger icon is only 20x20px. WCAG requires touch targets to be at least 44x44px. Only visible on mobile (<768px).

Bug 3: Fixed-width element causes horizontal scroll on mobile

This element is 500px wide with no max-width or responsive sizing. On screens narrower than 500px, it causes horizontal scrolling.

Bug 4: Media query gap (invisible at 375-400px)

This yellow section disappears completely on viewports between 375px and 400px wide. It's visible at 374px and below, and at 401px and above, but not in between.

Bug 5: Font size too small for mobile (10px)

This paragraph uses a 10px font size, which is far below the recommended minimum of 16px for mobile devices. Users on mobile will need to zoom in to read this text, which creates a poor user experience. The recommended minimum font size for body text on mobile is 16px to ensure readability.

Bug 6: Button grid doesn't wrap on mobile (flex-wrap: nowrap)

These buttons use flex-wrap: nowrap, so they overflow horizontally on narrow screens instead of wrapping.
