Bug 1: PageView Event — Missing page_title

Bug — missing field

A PageView event is fired to POST /api/track on page load, but the payload omits the required page_title field. The server will accept it, but downstream analytics will show null titles.
Payload sent: { event: "PageView", page_url: "...", timestamp: "..." }

Bug 2: "Buy Now" Click — Wrong element_id

Bug — wrong field value

Clicking "Buy Now" fires a click event to POST /api/track, but sends element_id: "btn-123" instead of the correct "buy-btn". The actual DOM id of the button is buy-btn.

Bug 3: Duplicate Conversion Pixel

Bug — duplicate request

A conversion pixel fires immediately: /pixel.gif?event=purchase&value=99. Then, 500 ms later, the exact same pixel fires again. Both requests appear in the Network tab.

Pixel fired on page load — check Network tab for two identical requests ~500 ms apart.

Bug 4: "Subscribe" Button — Empty Payload

Bug — empty payload

Clicking Subscribe fires a tracking event to POST /api/track with an empty object {} as the body. No event name, no user data, no context.

Bug 5: Pixel with Missing user_id Value

Bug — malformed params

A signup pixel fires with user_id= (empty value): /pixel.gif?event=signup&user_id=. The parameter is present but has no value — the server will record an empty string.

Pixel fired on page load — check Network tab for /pixel.gif?event=signup&user_id=.

Bug 6: Scroll Section — Missing IntersectionObserver

Bug — no scroll event fired

The element below should fire a SectionVisible tracking event when it enters the viewport, but the IntersectionObserver code is absent. Scrolling down to reveal this section produces no network request.

Featured Section Scroll tracking should fire here — but no IntersectionObserver is attached.
